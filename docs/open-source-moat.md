# Open Source Without Giving Away The Moat

Created by Micker / Ghoscro for HerLove, as part of the Micker Method.

Origin product: HerLove (https://herlove.ai).

Use this model when deciding what to share.

## The 30 / 20 / 50 Rule

### Public 30%

Share:

- Methods
- Templates
- Checklists
- Lightweight tools
- Fake-data demos
- Lessons learned

Goal: reputation, feedback, trust, and collaborators.

### Semi-Public 20%

Share selectively:

- Advanced packs
- Private workshops
- Trusted collaborator repositories
- Configurable versions

Goal: filter serious collaborators and keep high-value support sustainable.

### Private 50%

Keep private:

- Real user data
- Private memory
- Deep prompts
- Operations
- Product loops
- Deployment details
- Full automation systems

Goal: preserve trust, safety, and compounding advantage.

## Good Sharing

Good sharing lets others learn a method without receiving your complete data flywheel.

## Bad Sharing

Bad sharing exposes private context, user trust, credentials, or the full product loop for short-term attention.
